# 正确认识AI#

## AI的定位##

有人会觉得只要会问问题，就能一步到位解决科研问题，或者觉得AI可以提供任何信息。但是其实AI只是搜集了大量的网络信息进行整合学习再根据提问进行输出回答。

我们需要明确各种AI工具是辅助人的工作，帮助我们提升工作学习效率的，不能完全代替人的思考。把AI看作**听从指令完成工作的下属**会更加恰当。它可以快速完成你交代的任务，但不会主动理解你的研究目标，也不会替你判断研究方向的合理性。

真正决定科研质量的，仍然是“人如何思考问题”，而不是“AI有多强”。在整个过程中，**AI负责执行，你负责思考与决策**。

## AI适合做什么##

AI擅长：

- 模式化任务
- 信息整理
- 代码生成
- 语言表达优化

AI不擅长：

- 理解你的真实研究意图
- 判断数据是否合理
- 自动发现你的实验设计问题
- 替你做科学决策

在实际问题中，AI无法探索到你真正的意图和进度，需要给它提供更加精准的信息才能得到更合适高效的回答。

## AI的主要能力##

| 任务类型     | AI可以提供的帮助                               |
| ------------ | ---------------------------------------------- |
| 代码编写     | 快速生成基础代码框架（数据处理、建模、可视化） |
| 报错调试     | 分析报错原因、定位问题层级、提供修改建议       |
| 文献理解     | 总结论文核心内容、提取方法与结论               |
| 数据分析     | 辅助进行特征分析、统计思路建议                 |
| 论文写作     | 英文润色、结构优化、表达规范化                 |
| 图表与可视化 | 生成流程图、方法框架图、科研示意图             |
| Prompt设计   | 优化提示词结构，提高AI输出稳定性               |

使用AI的本质不是提问而是下达任务，把AI看作执行力更强知识面更广的执行团队，给AI的任务越明确清晰，就能得到越高效稳定的回答。

# 工具篇

## 必备工具##

1. Python环境：Anaconda
2. 编译器：VsCode
3. Markdown：Typora（有免费版）
4. 写论文：Overleaf网页版

## Python环境配置##

可以不用下载Python官方版本，直接选择安装Anaconda就行，已经包含有Python。

在vscode里选择conda环境作为Python Interpreter，后续在VSCode中进行代码编写即可。

有另外需要下载的库在终端操作即可。

> 常见问题：由于python版本不一致导致代码出错。配置好环境后确认python的版本即可有效规避。

## 如何调用API

标准调用流程如下：

1. 在大模型网页找到获取 API Key（国内Qwen网页版有专门一栏为API，创建API保存好API Key即可）
2. 在VSCode终端安装对应SDK（如 openai / anthropic）
3. 在代码中配置 key
4. 发起请求并获取返回结果

## 国内如何使用ChatGPT等国外大模型##

首先需要配置vpn环境，用Chrome浏览器搜索官网。可以选择注册谷歌邮箱，用谷歌邮箱登录获取自己的账号，也可以选择在淘宝上购买成品号。

还有一个方法就是用类似与模型中转站的网站，但需要注意鉴别是否可靠。优点是更加方便一个网站集结了多种模型，封号的风险低，坏处就是可能会降智。

## 如何更好地使用AI辅助编程##

不要只和AI说：
> 代码有错误

要明确告诉它：
> 具体报错是什么
> 你的python版本是什么
> 代码这部分是怎么写的
> 目标是什么（比如修复错误、解释原因、以后应该如何避免等）

# 实战篇#

## 为什么AI总是出错##

检查提问时：

- 是否有明确的**指令目标**给到它告诉它具体需要做什么 
- 是否提供了**完整明晰的信息**
- 是否给到了合适完整的**约束**
- 是不是没有进行**任务拆解**，而是一整个庞大的任务让它一下子完成

要知道AI无法理解你的想法目的，只会执行你写出来的规则命令。

## 如何写Prompt

首先要认识到prompt不是提问的问题，而是给AI的**任务说明书**。不是不能向AI提问，而是提问也应该是任务命令式的提问。

参考结构：

- 背景：任务场景
- 输入：数据 / 代码 / 报错
- 目标：希望得到什么结果
- 约束：格式 / 风格 / 限制
- 输出：希望AI输出什么形式

给AI的要求越明确，得到的结果就越准确稳定。

## Debug Prompt怎么写##

- 角色设定（如：你是一个xx方面的专家）
- 提供电脑系统、python版本信息
- 已经安装的库
- 定位错误类型（环境 / 代码 / 逻辑）
- 缩小问题范围
- 已经尝试但失败的方法
- 提供可验证修复方案
- 给出具体目标、要求输出的格式（如.md、.json）

# 进阶篇

## AI适合作什么图

擅长：

- 方法流程图
- 模型结构图
- 数据处理流程
- Graphical Abstract

不适合：

- 实验结果图（ROC / AUC）
- 精确数值图表
- 统计分析图

总结来说就是AI作图只适合逻辑的可视化，而不是数据、结果的可视化。

## 作图Prompt##

指定：

- 语言：中文/英文
- 风格：如商业风、严谨科技感
- 类型：如工作流程图
- 限制：如不要展示具体数据结果

```markdown id="pmt4"
## AI绘图 Prompt示例

请生成一张医学AI科研流程图。

要求如下：
- Language: English
- Style: clean, scientific, medical AI visualization
- Content: workflow only
- Must NOT include:
  - experimental results
  - numerical values
  - model performance metrics

Structure:
Data → Preprocessing → Model → Evaluation → Interpretation → Output
```

## 防止AI幻觉绘图的方法##

AI绘图最大问题不是“画得不好”，而是**“画错内容”**——即出现不属于你科研设计的结构、流程或变量。

因此核心目标不是优化图的美观，而是控制AI只做**结构表达**，不做内容创造。具体可以参考以下方法：

1. 在prompt中预先定义结构框架，避免AI自行增加模块。

   > 示例：Follow exactly this pipeline:
   >
   > Input → Preprocessing → Model → Output
   >
   > Do not add any extra steps.

2. 明确告诉AI只能使用哪些内容，防止AI编造不存在的内容到图片

   > 示例：You are only allowed to use the following components:
   >
   > - Data Input
   > - Feature Engineering
   > - Model Training
   > - Prediction Output
   >
   > Do not introduce new components.

3. 禁止AI过度解释扩展逻辑

   > 示例：Do not expand, modify, or interpret the pipeline.Only visualize what is explicitly provided.

4. 要求把输入当作最终结构定义而非描述，避免AI自由想象设计

   > 示例：Use the following exact structure:
   >
   > EHR Data → Cleaning → Feature Extraction → Logistic Regression → Evaluation

5. 禁止AI自动引用未提到的模型结构

   > 示例：Do not introduce any deep learning architectures (e.g., CNN, Transformer, LSTM).
   >
   > Only use the methods explicitly listed.

6. 要求AI在生成后自检结构一致性，提升输出稳定性

   > 示例：After generating the diagram, verify:
   >
   > - No extra nodes added
   > - No missing steps
   > - No invented components
   >
   > If any issue exists, regenerate.
